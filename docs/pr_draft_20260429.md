## Summary

This PR turns the repository into a three-benchmark debug-stage experiment
pipeline for the minimax queueing project.

Implemented and documented:

- routing BVI / AMQ / NNQ comparison diagnostics;
- routing normalized AMQ and fitted AMQ calibration analysis;
- service-rate-control BVI / AMQ / NNQ / NNQ+state0 guard comparisons;
- service-rate eval50 robustness check;
- service-rate NNQ failure and repair diagnostics;
- polling benchmark smoke loop;
- polling policy inspection and policy-shape diagnostics;
- polling NNQ `polling_augmented` state features;
- current project handoff for follow-up agents.

## Key Results

### Routing

Routing is currently the most mature benchmark.

- normalized AMQ is the selected AMQ performance baseline;
- fitted AMQ improves policy-shape diagnostics but loses important
  BVI-attacker defense mass;
- fitted AMQ remains a diagnostic condition, not the selected performance
  result.

Main handoff/report files:

- `docs/routing_amq_calibration_handoff.md`
- `results/routing_narrative_report.md`

### Service-Rate-Control

Service-rate-control has a solid debug-stage comparison.

Eval50 mean costs:

- BVI: `0.242748`
- AMQ: `0.293975`
- NNQ+state0 guard: `0.317562`
- raw NNQ: `0.460424`

Conclusion:

- raw NNQ is weak because it over-services at the empty queue;
- `state == 0 -> low service` repairs much of that failure;
- guarded NNQ remains a diagnostic/corrected baseline;
- BVI and AMQ remain stronger by mean cost under eval50.

Main files:

- `docs/service_rate_control_consolidation.md`
- `docs/service_rate_eval_robustness.md`
- `docs/service_rate_nnq_repair_diagnostic.md`

### Polling

Polling now has a runnable and diagnosable smoke loop.

Current best polling diagnostic comparison:

`results/polling_augmented_smoke_comparison/20260428T155107Z/summary.json`

Policy-shape summary:

| Method | Avg cost | Mean p_defend | Gap states defended | Shape |
|---|---:|---:|---:|---|
| BVI | `0.422117` | `0.291427` | `12 / 24` | state-dependent |
| AMQ | `0.294500` | `1.000000` | `24 / 24` | always-defend |
| NNQ augmented | `0.312392` | `0.491413` | `12 / 24` | state-dependent |

Conclusion:

- polling smoke loop is runnable and diagnosable;
- NNQ has a credible non-degenerate polling candidate via
  `state_feature_set: polling_augmented`;
- AMQ remains degenerate and should not be promoted to a polling result yet;
- polling should not be expanded to multiseed until AMQ is diagnosed.

Main files:

- `docs/polling_minimal_closure.md`
- `docs/polling_policy_calibration.md`
- `results/polling_augmented_policy_shape_diagnostic.md`
